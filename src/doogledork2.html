Identification & Analysis: Finding the Evidence

The identification phase is a hunt for digital breadcrumbs. Evidence resides across various system components, from volatile memory to the filesystem and registry. Understanding where to look and what tools to use is paramount to uncovering adversary activity.

Typical Evidence Distribution

Analysis shows that forensic artifacts are not evenly distributed. Filesystem and Registry artifacts often provide the most persistent evidence of execution and persistence mechanisms.

Normal Windows Processes vs. The Imposters

Knowing what's normal is the first step to finding evil. Attackers often name malware after legitimate system processes to blend in. Pay close attention to parent processes, user accounts, and image paths.

Process Normal Parent Normal User Suspicious Indicator
svchost.exe services.exe SYSTEM, Network Service, Local Service Running from a non-system directory, has network connections to unusual IPs.
lsass.exe wininit.exe SYSTEM Multiple instances running, or parented by any process other than wininit.exe.
csrss.exe Orphan Process (smss.exe exits) SYSTEM Parent process is not smss.exe, unusual modules loaded.
explorer.exe Orphan Process (userinit.exe exits) Logged-on User Running as SYSTEM, or parented by a process other than userinit.exe.

Following the Trail: Lateral Movement

Adversaries don't stand still. Once they gain a foothold, their goal is to move across the network to find valuable data or gain higher privileges. Detecting the artifacts left by these lateral movement techniques is key to scoping an intrusion.

Remote Desktop (RDP)

Artifacts: Security Event Logs (ID 4624, Logon Type 10), RDP Bitmap Cache files, and MRU entries in the user's NTUSER.DAT hive.

PsExec & Remote Services

Artifacts: Service creation events (Event ID 7045), execution of PSEXESVC.exe, and presence of the executable in Prefetch and ShimCache.

Windows Management (WMI)

Artifacts: Execution of `WmiPrvSE.exe` as a child of `svchost.exe`, and analysis of WMI repository files (`OBJECTS.DATA`).

Scheduled Tasks

Artifacts: Task creation events (Event ID 4698), XML task files in `System32\Tasks`, and registry keys under `HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Schedule`.

Mapped Network Shares

Artifacts: `net use` command execution history, `MountPoints2` key in NTUSER.DAT, and LNK files pointing to remote shares.

PowerShell Remoting

Artifacts: PowerShell logs (Event IDs 4103, 4104), network connections on ports 5985/5986, and execution of `wsmprovhost.exe`.

Unpacking the Payload: Malicious Document Analysis

Phishing with malicious documents remains a primary initial access vector. Analyzing these files involves a methodical process of identifying risky components, extracting embedded code, and deobfuscating the payload to understand its true intent.

📄

1. Identify Risky Keywords

Use tools like `pdfid.py` or `olevba.py` to scan for keywords like `/JavaScript`, `/OpenAction`, or `Sub AutoOpen()` that indicate macros or scripts.

📥

2. Extract Embedded Code

Use tools like `oledump.py` or `rtfobj.py` to extract the suspicious macro, JavaScript, or shellcode streams from the document file.

🔦

3. Deobfuscate & Analyze

Manually or with tools like `vmonkey` or debuggers, deobfuscate the code to reveal its true functionality, such as downloading the next stage malware.
