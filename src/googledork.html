Reconnaissance: Google Dorking

Google Dorking, or Google Hacking, is a crucial passive reconnaissance technique. It involves using advanced search operators to find specific information that isn't readily available through simple searches. This module will introduce you to the core operators and provide an interactive lab where you can practice constructing dorks to uncover potentially sensitive information, such as exposed configuration files, login pages, and vulnerable directories.

Common Dorking Operators

Operator Description Example
intitle:Searches for pages with specific text in the title.intitle:"index of"
inurl:Searches for pages with specific text in the URL.inurl:admin.php
filetype:Restricts search to specific file types.filetype:sql "password"
site:Limits search to a specific website.site:example.com
intext:Searches for text within the body of a page.intext:"confidential"

Web Application Attacks

Understanding common web application vulnerabilities is fundamental for both defense and incident response. This module explores two of the most persistent threats: Cross-Site Scripting (XSS) and Cross-Site Request Forgery (CSRF). Through interactive cards, you can explore the different types of each attack, see simplified examples of how they are exploited, and learn the key strategies required to mitigate them effectively.

Reflected XSS

Malicious script is reflected off a web server to the victim's browser, typically via a malicious link.

Example Payload:

<script>alert('XSS')</script>

Mitigation:

Input validation and context-aware output encoding are critical.

Stored XSS

Malicious script is permanently stored on the target server (e.g., in a database via a comment field).

Attack Vector:

User submits a comment with an embedded script. Every user viewing the comment is affected.

Mitigation:

Sanitize all user-supplied data before storing it and encode it upon display.

DOM-based XSS

The vulnerability exists in client-side code, where data is manipulated in an unsafe way in the DOM.

Example Sink:

document.write(location.hash)

Mitigation:

Avoid using unsafe JavaScript methods. Use trusted libraries for data handling.

CSRF Overview

An attack that forces an authenticated user to submit an unwanted request to a web application they are currently logged into.

Analogy:

Tricking someone into signing a document without them knowing what they are signing, using their existing authority.

Primary Defense

The most common and effective defense is the use of Anti-CSRF Tokens.

How it works:

A unique, secret token is embedded in each form. The server validates this token with every state-changing request, ensuring the request came from the original site.

SameSite Cookies

A modern browser defense that restricts when cookies are sent with cross-site requests.

Modes:

Strict: Prevents the cookie from being sent on all cross-site requests. Lax: Allows cookies with top-level GET requests. This is the browser default.
